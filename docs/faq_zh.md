# 常见问题解答 (FAQ)

## 一般问题

### 什么是维吾尔语分析器插件？
维吾尔语分析器插件是一个专为Elasticsearch设计的插件，提供高级的维吾尔语文本处理和分析功能。它包含统一词典系统、形态学分析器和多种分析视图，支持现代和古典维吾尔语文本的智能处理。

### 插件的主要功能有哪些？
- **统一词典系统**：管理多个词典来源（THU数据集+自定义词典）
- **形态学分析**：基于规则的智能词汇分析和还原
- **多视图分析**：原始视图、分割视图、自定义视图
- **ES集成**：完整的Elasticsearch分析器和令牌过滤器
- **优先级处理**：自定义词典优先，多层次分析策略

### 支持哪些维吾尔语文本类型？
- **现代维吾尔语**：当代书面语和口语词汇
- **古典维吾尔语**：通过元音弱化还原支持历史文本
- **专业术语**：科技、医学、教育等领域词汇
- **新词汇**：通过自定义词典支持现代术语

## 词典系统

### 词典系统是如何工作的？
插件使用统一词典管理器，包含两个主要词典：
1. **自定义词典**（最高优先级）：用户定义的现代词汇
2. **THU词典**：清华大学的21,000+形态学词汇数据

系统进行**词汇识别和形态还原**，而不是简单的分词。

### THU词典的格式是什么？
THU词典包含多种格式：
```
# 基本形式
ئا، ئائورتا

# 元音弱化还原
يېزىش.(يازىش) قا
# 现代形式: يېزىش，原始形式: يازىش，后缀: قا

# 复杂后缀组合
ئائىلى.(ئائىلە) سى دىكى لەر نىڭ

# 复数变位
ئورۇن. لىر(لار) ى نىڭ
```

### 如何添加自定义词汇？
1. 编辑 `src/main/resources/dictionaries/custom_dictionary.txt`
2. 添加词汇（基本形式和变位形式）：
   ```
   # 新词汇
   تېخنىكا
   تېخنىكا. لار
   تېخنىكا. نىڭ
   ```
3. 重新构建：`./gradlew clean check`

### 自定义词典的优先级如何？
自定义词典具有**最高优先级**（95%置信度），会优先于THU词典进行匹配。这确保了用户定义的现代术语能够被正确识别。

## 形态学分析

### 什么是形态学分析？
形态学分析是对维吾尔语词汇进行结构分析的过程，包括：
- **词根识别**：找到词汇的基本形式
- **后缀分析**：识别语法后缀和其功能
- **元音弱化还原**：恢复历史语言形式
- **置信度评估**：提供分析结果的可信度

### 分析器的策略层次是什么？
系统采用多层次分析策略：
1. **自定义词典精确匹配**（95%置信度）
2. **THU词典精确匹配**（95%置信度）
3. **THU词典部分匹配**（85%置信度）
4. **基于规则的分析**（75%置信度）
5. **统计预测分析**（65%置信度）
6. **回退策略**（30%置信度）

### 如何处理未知词汇？
对于词典中不存在的词汇，系统会：
1. 应用形态学规则进行分析
2. 使用后缀模式匹配
3. 基于统计信息进行预测
4. 提供最佳猜测结果和置信度

## Elasticsearch集成

### 如何在ES中配置分析器？
```json
{
  "settings": {
    "analysis": {
      "analyzer": {
        "uyghur_original": {
          "type": "uyghur_original_analyzer"
        },
        "uyghur_split": {
          "type": "uyghur_split_analyzer"
        }
      }
    }
  }
}
```

### 原始分析器和分割分析器的区别？
- **原始分析器**：提供元音弱化还原后的历史形式，适用于学术研究
- **分割分析器**：保持现代书写形式，适用于当代文本处理

### 如何测试分析效果？
```bash
# 测试原始分析器
curl -X POST "localhost:9200/_analyze" -d '{
  "analyzer": "uyghur_original_analyzer",
  "text": "يېزىشقا كىتابلارنىڭ"
}'

# 测试分割分析器
curl -X POST "localhost:9200/_analyze" -d '{
  "analyzer": "uyghur_split_analyzer",
  "text": "يېزىشقا كىتابلارنىڭ"
}'
```

## 安装和部署

### 系统要求是什么？
- **Elasticsearch**: 8.x 或 9.x
- **Java**: Elasticsearch 8.x 构建需要 Java 17 或更高版本；Elasticsearch 9.x 构建需要 Java 21 或更高版本
- **内存**: 建议至少512MB用于词典加载
- **存储**: 约50MB用于插件和词典文件

### 如何安装插件？
1. **构建插件**：
   ```bash
   ./gradlew clean check
   ```

2. **安装到ES**：
   ```bash
   elasticsearch-plugin install file:///path/to/plugin.zip
   ```

3. **重启Elasticsearch**

### 如何验证安装成功？
```bash
# 检查插件列表
elasticsearch-plugin list

# 测试分析器
curl -X POST "localhost:9200/_analyze" -d '{
  "analyzer": "uyghur_split_analyzer",
  "text": "سالام دۇنيا"
}'
```

### Docker环境如何部署？
```bash
# 复制插件到容器
docker cp plugin.zip es:/tmp/

# 进入容器安装
docker exec -it es elasticsearch-plugin install file:///tmp/plugin.zip

# 重启容器
docker restart es
```

## 性能和优化

### 插件的性能如何？
- **词典加载**：启动时一次性加载，约2-3秒
- **查找速度**：基于哈希表，平均O(1)时间复杂度
- **内存使用**：约30-50MB用于词典索引
- **分析速度**：每秒可处理数千个词汇

### 如何优化性能？
- **内存配置**：为ES分配足够内存
- **词典优化**：将常用词汇添加到自定义词典
- **缓存策略**：系统自动缓存频繁查找的结果
- **批量处理**：使用bulk API进行大量文本处理

### 大规模部署注意事项？
- 在所有ES节点上安装插件
- 确保词典文件在所有节点上同步
- 监控内存使用情况
- 定期更新自定义词典

## 故障排除

### 常见错误及解决方法

#### 插件安装失败
```
错误：java.nio.file.AccessDeniedException
解决：使用root权限或检查文件权限
```

#### 词典加载失败
```
错误：Dictionary file not found
解决：检查词典文件路径和权限
```

#### 分析器不工作
```
错误：Unknown analyzer type
解决：确认插件正确安装并重启ES
```
