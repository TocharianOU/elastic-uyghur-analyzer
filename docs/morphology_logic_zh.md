# 形态分析底层原理简述

本项目的维吾尔语分析器不是简单按空格或固定后缀切词，而是结合词典、词缀清单、权重模型和元音和谐规则进行形态分析。

## 两种输出视图

分析器支持两种主要视图：

- `SPLIT`：保留现代书写形式，适合搜索、索引和用户输入匹配。
- `ORIGINAL`：尽量还原弱化前的规范形式，适合形态学分析和词干归一化。

例如词典中存在：

```text
ئائىلى.(ئائىلە) دىكى
```

则 `SPLIT` 输出现代写形：

```text
ئائىلى + دىكى
```

`ORIGINAL` 输出规范形：

```text
ئائىلە + دىكى
```

## 分析优先级

分析器按以下顺序处理输入词：

1. 优先查自定义词典和 THUUyMorph 词典的完整匹配。
2. 如果没有完整匹配，查最长词典前缀，再分析剩余后缀链。
3. 如果词典路径不足，使用基于词典统计的 Viterbi 权重模型寻找较优切分。
4. 对未登录词使用后缀回退规则进行 OOV 切分。
5. 如果仍无法可靠分析，则返回整词。

## 后缀与附着词

后缀和附着词来自结构化 TSV 词缀清单。分析时优先识别最外层附着词，再处理普通后缀。

TSV 中带有歧义警告的词缀不会被盲目剥离，避免把派生词或同形词误切开。

## 元音和谐

当前新增的和谐分类器按最后一个非中性元音判断词干类别：

```text
BACK:    ا و ۇ
FRONT:   ە ۆ ۈ
NEUTRAL: ى ې
```

`ى` 和 `ې` 被视为中性元音，不直接决定和谐类别。如果一个词只有中性元音，当前默认按 `FRONT` 处理。

## 规范形还原

`ORIGINAL` 视图下会做两类还原：

- 词干还原：如 `ئائىلى -> ئائىلە`
- 后缀还原：如 `لىر -> لەر` 或 `لىر -> لار`

后缀还原来自 THUUyMorph 中的括号标注，例如：

```text
لىر(لەر)
```

分析器会把括号外作为现代写形，把括号内作为规范形，并结合词干的元音和谐类别选择合适的规范后缀。

## 总体原则

当前逻辑可以概括为：

```text
词典优先 + 词缀清单约束 + 权重模型补充 + OOV 后缀回退 + ORIGINAL 规范形还原
```

这样既保留现代书写形对搜索的友好性，也能在需要时输出更接近形态学规范的分析结果。
