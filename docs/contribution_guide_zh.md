# 贡献指南

感谢您对 Elasticsearch 维吾尔语分析器插件的关注！本指南将帮助您开始为项目做出贡献。

## 开始贡献

### 前提条件

- Elasticsearch 8.x 构建需要 JDK 17 或更高版本
- Elasticsearch 9.x 构建需要 JDK 21 或更高版本
- Git
- 基本的 Elasticsearch 插件知识
- 熟悉 Java 和 Gradle

### 开发环境设置

1. **Fork 并克隆**：
   ```bash
   git clone https://github.com/your-username/elastic-uyghur-analyzer.git
   cd elastic-uyghur-analyzer
   ```

2. **构建项目**：
   ```bash
   ./gradlew clean check
   ```

3. **运行测试**：
   ```bash
   ./gradlew test
   ```

## 贡献方式

### 1. 词典增强

插件使用位于 `src/main/resources/dictionaries/` 的几个词典文件：
- `custom_dictionary.txt`：用户自定义词汇（最高优先级）
- `thuuy_morph_raw.txt`：清华形态学数据集

**如何贡献**：
- 向 `custom_dictionary.txt` 添加新词汇条目
- 遵循格式：`词汇. 词根 后缀1 后缀2`
- 使用形态学分析器测试您的添加

### 2. 代码改进

#### 增强领域：
- **性能优化** `RuleBasedMorphologyAnalyzer`
- **新分析策略** 具有不同置信度级别
- **错误修复** 形态学分析逻辑
- **新词单元过滤器** 用于特定用例

#### 代码规范：
- 遵循 Java 命名约定
- 为公共方法添加 JavaDoc 注释
- 为新功能包含单元测试
- 保持向后兼容性

### 3. 测试和质量保证

- 测试受支持的 Elasticsearch 8.x 与 9.x 版本
- 为维吾尔语形态学边缘情况添加测试用例
- 使用大型文本语料库进行性能测试
- 真实场景的集成测试

### 4. 文档

- 为新功能更新 README 文件
- 在 `docs/` 目录中添加示例
- 改进代码注释和 JavaDoc
- 为特定用例创建教程

## 开发工作流程

### 1. 问题创建

开始工作前：
- 检查 GitHub 上的现有问题
- 创建新问题描述问题或功能
- 与维护者讨论方法

### 2. 分支策略

```bash
# 创建功能分支
git checkout -b feature/your-feature-name

# 或用于错误修复
git checkout -b fix/issue-description
```

### 3. 代码更改

- 使用清晰消息进行集中提交
- 遵循约定提交格式：
  ```
  feat: 添加新的形态学分析策略
  fix: 解决词典解析问题
  docs: 更新安装指南
  ```

### 4. 测试

提交前：
```bash
# 运行所有测试
./gradlew test

# 构建插件
./gradlew clean check

# 使用 Elasticsearch 测试（可选）
# 按照构建指南在 Docker 中测试
```

### 5. 拉取请求

1. **推送您的分支**：
   ```bash
   git push origin feature/your-feature-name
   ```

2. **创建拉取请求**：
   - 使用清晰的标题和描述
   - 引用相关问题
   - 包含测试说明
   - 如适用，添加截图

3. **PR 模板**：
   ```markdown
   ## 描述
   更改的简要描述

   ## 更改类型
   - [ ] 错误修复
   - [ ] 新功能
   - [ ] 文档更新
   - [ ] 性能改进

   ## 测试
   - [ ] 测试在本地通过
   - [ ] 为更改添加了新测试
   - [ ] 使用 Elasticsearch 测试

   ## 相关问题
   修复 #问题编号
   ```

## 代码审查流程

1. **自动检查**：所有 PR 必须通过 CI/CD 检查
2. **同行审查**：至少需要一个维护者审查
3. **测试**：更改必须包含适当的测试
4. **文档**：为面向用户的更改更新文档
